# CLI 录屏自动化：口播稿 → 录屏脚本 → MP4

输入一篇口播稿（或录屏指示），自动生成 bash 录屏脚本。用户在终端跑脚本，输出 1080p MP4。

## 核心理念

**口播稿即脚本蓝图。** 每段口播对应一个屏幕操作（命令/静态等待/展示），从口播稿里提取出命令序列和时长，填入录制模板。

**看到什么录什么。** ffmpeg 录物理屏幕 + tmux 前台显示 Claude Code，不需要 osascript/辅助功能权限。

## 工作流

### 1. 输入口播稿

接收用户提供的：
- 口播稿文件路径（`script.md`）
- 或录屏指示文件路径（`录屏指示.md`）
- 或直接描述要录制什么

如果用户提供了口播稿但没有录屏指示 → 根据口播稿内容自动推断每段对应的 Claude Code 命令。
如果用户提供了录屏指示 → 直接用指示里的命令序列。

- **完成条件：** 拿到完整的命令序列和每步时长

### 2. 解析口播稿→命令序列

从口播稿/录屏指示中提取：

| 口播段落 | 屏幕操作 | 脚本动作 |
|---------|---------|---------|
| 开场白/过渡/结尾（无操作描述） | 静态画面 | `sleep N` |
| "说「抓今天的AI精选」" | 执行命令 | `send_cmd "抓今天的AI精选"` |
| "说「帮我写口播稿」" | 执行命令 | `send_cmd "帮我写口播稿"` |
| "展示 output/ 文件夹" | 查看命令 | `send_cmd "ls output/"` |
| "npx skills add..." | 安装演示 | `send_cmd "npx skills add..."` |

提取规则：
- 引号里的内容（「」或""）→ 直接作为 send_cmd 的参数
- 无引号但有动词（展示/运行/输入）→ 推断命令
- 纯叙述段落 → 按口播时长估算 sleep 秒数（正常语速约 3 字/秒）
- 需要等 Claude Code 完成的 → 加 `wait_for_claude`
- 只需展示几秒的 → 加 `sleep`

- **完成条件：** 生成结构化的命令列表，每条含命令文本、等待方式、标签

### 3. 环境检测

```bash
which ffmpeg && which tmux
ffmpeg -f avfoundation -list_devices true -i "" 2>&1 | grep "Capture screen"
```

- 缺工具 → 提示安装
- 多屏幕 → 让用户确认索引（`ffmpeg -y -f avfoundation -i "N" -t 3 /tmp/test.mp4` 逐个测试）
- **完成条件：** 确认屏幕索引

### 4. 生成录制脚本

基于 `scripts/record_template.sh` 模板生成。模板包含：
- ffmpeg 录屏启动/停止
- tmux 会话管理
- `send_cmd()` 和 `wait_for_claude()` 函数
- 1080p 转码逻辑

只需填充：**命令序列块**（第3步提取的）。

生成脚本写到用户指定目录（或当前目录），文件名格式：`record_<教程名>.sh`

### 5. 交付+指导

输出：
1. 生成的 `record_xxx.sh` 脚本
2. 运行命令：`bash record_xxx.sh`
3. 提醒：
   - 终端全屏（Cmd+Enter）
   - 开勿扰模式
   - 按回车后别碰键盘鼠标
   - Gate 确认框按回车继续

**不自动执行脚本** — 录屏需要用户手动在终端启动。

## 操作规则

- **口播稿驱动** — 每个命令都能追溯到口播稿的某一段
- **不自动执行录屏脚本** — 涉及屏幕接管，必须用户手动运行
- **屏幕索引需确认** — 不同机器配置不同，不能硬编码
- **wait_for_claude 用于 Claude 执行命令** — 普通展示（ls/cat）用 sleep 就够
- **静态段落用 sleep** — 开场/过渡/结尾等无操作段落，按口播字数÷3估算秒数

## 不适用场景

- 录制桌面应用（非终端） → 用 OBS
- 带声音解说 → 需额外配置音频输入
- 实时直播 → 这是录制方案不是直播方案

## 踩坑速查

| 问题 | 原因 | 解决 |
|------|------|------|
| 录错屏幕 | 多显示器索引不对 | `ffmpeg -list_devices` 确认 |
| tmux 里 claude 立即退出 | 网络/初始化偶发问题 | 重跑 |
| 画面静止 | claude 等待输入 | 手动按回车 |
| 视频太大 | CRF 值太低 | 改 `-crf 23` |
| 命令没发完 | 超时 | 增大 `max_wait` |

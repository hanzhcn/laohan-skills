# 教程型口播稿写作规范（jiaochengstyle）

> 版本: v1.0 | 2026-05-27 | 适用：寒武纪说AI 教程型视频
> 用途：大模型依据此规范 + 原始素材 → 直接输出合规口播稿，无需多轮修正
> **本文件完全自包含，不需要加载 style.md。** 选中此风格后只读这一份文件即可。

---

## 整体结构：五层模型

每篇教程型口播稿严格按以下五层结构组织，缺一不可：

```
┌─────────────────────────────────────┐
│ 1. 开头层（钩子+全覆盖+行动号召）     │  固定位置：全文最前
├─────────────────────────────────────┤
│ 2. 衔接段（大众误解→反常识正解）      │  固定位置：开头层之后、正文之前
├─────────────────────────────────────┤
│ 3. 正文层（N段，每段六要素+过渡）     │  主体内容，段数由话题决定
├─────────────────────────────────────┤
│ 4. 品牌层（真实分享+比喻类比）        │  位置灵活，嵌在正文层中
├─────────────────────────────────────┤
│ 5. 结尾层（总结+素材+收藏+ending）    │  固定位置：全文最后
└─────────────────────────────────────┘
```

---

## 第1层：开头层

**位置**：全文第一段
**功能**：5秒内抓住注意力 + 降低跳出 + 拉互动数据

必须包含3个成分，顺序固定：

| 成分 | 写法 | 作用 | 示例 |
|------|------|------|------|
| 痛点钩子 | 用2-3个问句浓缩目标受众的痛点 | 情绪共鸣 | "还在一个一个平台翻热点？还在百度搜问题翻三页全是广告？" |
| 全覆盖声明 | 一句话说"不管你是X还是Y都适合" | 降低跳出 | "从入门到进阶，不管你是纯小白还是老手都有适合你的" |
| 行动号召 | "点赞收藏，我们直接开始" | 拉互动数据 | 固定句式，每篇必须 |

**规则**：
- 禁止绕弯子。痛点钩子必须在第一句就出现
- 禁止先自我介绍再切入主题。身份信息放在结尾层
- 全覆盖声明必须具体指向受众分类，不能写"适合所有人"
- 如有真实分享预告，也在此层末尾说出："还会把XX分享给你"

---

## 第2层：衔接段

**位置**：开头层之后、正文第一段之前
**功能**：用认知颠覆稳住注意力，从情绪（钩子）过渡到理性（正文）

必须包含2个成分：

| 成分 | 写法 | 作用 |
|------|------|------|
| 大众误解 | "先说一个很多人没搞明白的事" 或 "很多人以为X，其实不是" | 暗示"你之前的理解是错的"，触发好奇 |
| 反常识正解 | 给出跟大众认知相反的正确答案 | 让观众获得"顿悟感"，建立信任 |

**规则**：
- 必须有反常识成分，不能是平淡陈述。"很多人以为X，其实不是" 是标准格式
- 衔接段内容必须跟全文主题直接相关
- 长度控制：3-5句话，说完立刻进正文

---

## 第3层：正文层

**位置**：衔接段之后、结尾层之前
**功能**：交付干货，每段一个独立主题

### 每段六要素（顺序灵活，但每个都必须有）

| # | 要素 | 定义 | 合格标准 |
|---|------|------|---------|
| 1 | 推荐 | 明确告诉观众推荐什么 | 观众读完能说出"他推荐的是X" |
| 2 | 痛点 | 用户在受什么苦 | 必须是具体场景。"五个平台看下来半小时没了"✓ "效率不高"✗ |
| 3 | 误区 | 用户常见的错误认知 | 格式："很多人以为X，其实不是" 或 "你可能觉得X，其实不是" |
| 4 | 如何用 | 具体操作步骤 | 观众看完能直接动手。用 `[字幕]` 标注命令/地址 |
| 5 | 好处 | 用了能得到什么 | 必须具体可感知。"十分钟全部看完"✓ "提升效率"✗ |
| 6 | 理由 | 凭什么这么推荐 | 必须有依据。"我自己实测下来" 或 "十几个平台接口都封装好了" |

### 过渡（每段结尾必须有）

- 用一句话把当前段跟下一段连起来
- 格式：先说当前段的局限/前提，再引出下一段要解决的问题
- 示例："但opencli只管拿数据，不管读内容。想读文章全文，看第二个。"

### 正文质量标准

| 标准 | 定义 | 反面案例 |
|------|------|---------|
| 干货 | 每句话都传递新信息，无重复 | 同一个好处说了两遍 |
| 丝滑 | 段间过渡有因果关系，不生硬 | "接下来讲第二个"✗ vs "但这个只管拿数据，看下一个"✓ |
| 节奏 | 快慢交替。痛点/误区快，好处/理由慢 | 全程一个语速 |

### B-Roll标注

所有B-Roll由NotebookLM根据口播稿生成PPT图片，后期可按需替换为录屏。

**格式**：`[B-Roll：幻灯片上要显示的文字内容]` — 口播不念，观众看到文字卡

**时长计算**：
```
纯口播字符 = 去掉标题、分隔线、空行、画面标注后的字符数
A-Roll秒数 = 纯口播字符 ÷ 8.2（校准源：老韩实测 8.2字符/秒）
B-Roll秒数 = A-Roll秒数 × 0.43（即B-Roll占总时长的30%）
总时长 = A-Roll秒数 + B-Roll秒数
B-Roll次数 = B-Roll秒数 ÷ 15（每次15秒，向下取整）
```

**插入规则**：
- 每个正文段的「如何用」处必须插一条B-Roll（展示命令/地址/操作步骤）
- 正文段出现对比、数据、方案列表时插一条
- 结尾总结段插一条
- 不要在开头层和品牌层真实分享处插入（需要真人的脸）

**叠加文字**：`[字幕：命令/地址]` — 口播不念，剪辑时叠加在A-Roll画面上

## 第4层：品牌层

**位置**：嵌在正文层中，位置灵活
**功能**：转粉 + 记忆点

### 真实分享（全篇至少1个）

| 属性 | 说明 |
|------|------|
| 定义 | "我怎么用的" — 创作者的真实使用场景 |
| 作用 | 建立信任 → 转粉 |
| 必须包含 | 具体场景 + 具体动作 + 具体结果 |
| 正确 | "老韩每天早上做选题，用opencli把知乎热榜抓一遍，十分钟全部看完" |
| 错误 | "我觉得这个工具很好用"（无场景无动作无结果）|
| 铺垫 | 开头层可预告"还会把XX分享给你"，正文中兑现 |

### 比喻/类比（全篇至少1个）

| 属性 | 说明 |
|------|------|
| 定义 | 用生活化场景让抽象概念秒懂 |
| 作用 | 降低理解门槛 + 创造记忆点 |
| 必须 | 观众日常能接触的东西，不能是另一个抽象概念 |
| 正确 | "Claude Code是框架，Claude是模型，就像手机和SIM卡" |
| 错误 | "它像一个中间件，把上下层解耦"（用术语解释术语）|

### 真实分享 vs 素材提供的区别

- 真实分享 = "我怎么用的" → 驱动信任 → 转粉
- 素材提供 = "我给你准备好了" → 驱动依赖 → 拉收藏
- 经常成对出现：先分享用法，再说"东西给你准备好了"

---

## 第5层：结尾层

**位置**：全文最后
**功能**：总结 + 拉收藏 + 固定收尾

必须包含4个成分，顺序固定：

| # | 成分 | 写法 |
|---|------|------|
| 1 | 一句话总结 | 浓缩成行动指令，不复述内容 |
| 2 | 素材提供 | "XX我全准备好了"（呼应开头预告） |
| 3 | 系列标识 | "这是「寒武纪说AI」的第X期" |
| 4 | 固定ending | "我是老韩，咱们下期见。" |

**规则**：
- 禁止重复正文内容
- 素材提供必须跟评论区置顶内容对应

```
> 系列名：寒武纪说AI | 教程型
> 预计时长：X分钟 | 约XXX字（口播）+ 画面标注

[口播稿正文]

## 拍摄备注
- **标题**：抖音发布标题
- **封面关键词**：2-3个
- **核心观点**：一句话
- **收藏引导**：结尾引导语
- **评论区置顶**：具体内容清单（命令、地址、模板）
- **系列标签**：寒武纪说AI
- **预计时长**：X分钟（约XXX字口播）
- **五层结构自检**：逐层确认完整性
- **画面标注说明**：`[画面]`=录屏/截图，`[字幕]`=叠加文字
```

## 自检清单（输出前必须逐项确认）

### 结构完整性

- [ ] 开头层：痛点钩子 + 全覆盖声明 + 行动号召
- [ ] 衔接段：大众误解 + 反常识正解
- [ ] 正文每段：推荐/痛点/误区/如何用/好处/理由，六要素全覆盖
- [ ] 正文每段：有过渡句
- [ ] 品牌层：≥1个真实分享（场景+动作+结果）
- [ ] 品牌层：≥1个比喻/类比（生活化）
- [ ] 结尾层：总结 + 素材提供 + 系列标识 + 固定ending

### 内容质量

- [ ] 无重复：同一信息不出现两次
- [ ] 痛点具体：每个都是可感知场景
- [ ] 误区有钩力：用"很多人以为X，其实不是"格式
- [ ] 好处可感知：观众能感知到具体收益
- [ ] 理由有依据：有实测/事实支撑

### 数据准确

- [ ] 工具名称拼写正确
- [ ] Star数/版本号等数据与实际一致
- [ ] 安装命令可执行
- [ ] GitHub地址有效

## 语言质量规则

### 禁区词（触碰即失败）

意味着什么、本质上、换句话说、不可否认、综上所述、总的来说、值得注意的是、不难发现、作为一个、让我们、不妨、试想一下、值得一提的是、赋能、抓手、赛道（比喻除外）、壁垒、痛点

### AI味检测（命中即失败）

- 体制化开头：不可否认/众所周知/在当今的时代/随着...的发展
- 空心强化：真正地/不得不说/坦率地说
- 三连排比：连续 3+ 相同句式
- 冒号堆砌：一段内 >3 个「：」后接列表
- 过度总结：由此可见/这充分说明/这告诉我们
- 假设复数："我们都知道"/"每个人都会"（应用"你"单数）
- AI填充：值得注意的是/毋庸置疑/显而易见
- 空洞大词：至关重要/开创性的

### 禁用标点

冒号→逗号，双引号→「」

### 禁用结构

"首先其次最后"、"在当今的时代"开头、bullet超3个、空泛工具名

### 语感规则

- **短句是命**：关键节奏点 ≤15 字，短句比例 ≥60%
- 3个短句后可插1个长句做解释，长句用"的/了/吧/啊"断句
- 多用逗号制造呼吸点
- 反问句每3-4段至少1个

### 废话感（每篇挑4种以上，不硬塞，不出现在开场和告别语中）

单字结巴"能能"、夸张重复"等等等等等"、三连"太太太"、情绪"哎呦喂""哎"、方言"杠杠滴"、命令"你听我的"、自问自答、自嘲、数字犹豫"没有3万有2万"、独立评价"可有点意思"、连续反问"是不是...是不是..."、确认词"对吧""你说呢"、程度词"真的""巨"、口语解释"就是..."

### 用词规则

- 自称：主称"我"和"老韩"，号召用"咱"
- 称呼听众：开头叫「朋友们」，中间用「你」「你们」「大家」
- 动作词：丢给/喂给/塞给/跑通
- 解释词：「就是」是最强的口语解释词

### 转场句式

**推荐**（交替使用，不连续用同一种）：
- 强互动：「你可能会问」「你猜怎么着」「你想想看」「是不是有点意思」
- 因果收束：「所以说」「也就是说」「你看」
- 关联跳跃：「说到这个」（每篇最多2次）
- 节奏切换：「好了不绕了」「先说结论」
- 直接跳：不用过渡词，下一段直接开新话题

**禁用**：
- 「接下来我们聊聊」「然后我要讲的是」→ 念大纲
- 「首先/其次/最后」→ 书面连接词

### 二创脱敏

- 结构尽量错位：论据顺序不跟原稿走（信息密集型素材允许按原顺序）
- 比喻/类比：原稿好比喻可保留标注引用，替换必须 ≥ 原版冲击力
- 老韩增量 ≥2 处：每处提供新信息/新视角/新类比/新领域连接，禁空泛总结
- 论证路径换血：同一组数据换顺序或换数据组合
- 核心原话可标注引用保留，每篇 ≤2 处，不伪装成自己的
- 底线：不整段搬运（>3句连续雷同）

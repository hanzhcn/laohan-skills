# 多平台改写 Skill

将一段口播稿或文章，改写为适合抖音、小红书、公众号三个平台的版本。

## 输入要求

用户需提供一段口播稿或文章原文。如果用户没有提供，主动要求用户提供原文。

## 改写规则

### 抖音版

- **格式**：短视频口播稿，保持口语化
- **字数**：必须达到原文的 60%-80%。具体操作：保留原文所有核心段落，只删过渡句和重复表达，不删观点、数据、步骤
- **开头**：必须用痛点/反常/好奇心钩子开头，禁止"大家好"式开场
- **结尾**：一句话收束 + 一个可操作步骤
- **禁止**：书面语、长句、排比句、空洞感叹
- **自检**：写完后统计字数，如果低于原文60%，说明删多了，补回被删的核心内容

### 小红书版

- **格式**：图文笔记（不含图片，只出文字内容）
- **标题**：单独一行，15字以内，用「」包裹
- **正文**：每段 1-2 句话，段落间空行
- **emoji**：每个要点前加 1 个相关 emoji，全文不超过 8 个
- **标签**：末尾加 3-5 个 #标签
- **禁止**：长段落、纯口号、过度堆砌 emoji

### 公众号版

- **格式**：深度长文，带小标题结构
- **字数**：必须达到原文的 120%-150%。具体操作：每个核心观点都要展开论述，增加具体场景描述、使用体验、数据解读，不能只罗列要点
- **标题**：单独一行，20字以内，有信息量（不要标题党）
- **结构**：开头引出问题 → 2-3 个核心观点（带小标题）→ 结尾总结 + 互动引导
- **禁止**：口语化表达、emoji、短视频节奏
- **自检**：写完后统计字数，如果低于原文120%，说明展开不够，回到每个小标题下补充具体论述

## 输出格式

用以下格式输出，三个版本之间用分隔线：

```
## 抖音版
[改写后的口播稿]

---

## 小红书版
「[标题]」

[改写后的图文笔记正文]

#标签1 #标签2 #标签3

---

## 公众号版
[标题]

[改写后的长文]
```

## 注意事项

- 核心信息（观点、数据、结论）三个版本必须一致，只是表达方式不同
- 不要在任何一个版本中凭空增加原文没有的信息
- 如果原文太短（少于100字），主动提醒用户"原文较短，公众号版可能展开不够充分"

# Rubric Notes — 教程型视频（Tutorial-Based Video）v2.0 bump-calibrated

> 基于 拉斐尔2077 的 40 条样本反向验证 + 5 条自有视频 BUMP 校准
> v2.0 改动：EV 合并进 OP（减少维度）、HP 权重 10%→15%（传播力被低估）、EC 权重 10%→5%（与 CV 部分重叠）、DF 硬门槛分两档
> DeepSeek 跨模型独立审核通过

---

## v2.0 综合分公式

```
composite = TS×0.20 + OP×0.20 + DF×0.15 + CV×0.25 + HP×0.15 + EC×0.05
```

**DF 硬门槛（分两档）**：
- DF ≤ 3（视频 ≤ 90 秒）→ composite 封顶 5.0
- DF = 4（视频 91-120 秒）→ composite 封顶 6.0
- DF ≥ 5（视频 > 120 秒）→ 正常计算

每个维度 0-10 整数分。综合分范围 0-10。

**权重逻辑**：
- **核心内容维度占 65%**：TS(20%) + OP(20%) + CV(25%)
- **传播维度占 20%**：HP(15%) + EC(5%)
- **DF(15%)**：独立维度，时长不足是硬伤

**v1.4→v2.0 改动**：
- EV（效果验证）合并进 OP（可操作性），OP 权重从 15% → 20%
- HP 从 10% → 15%（5 条数据证明钩子是传播力最强预测因子）
- EC 从 10% → 5%（资源钩子/收藏引导已在 CV 里评估，不需要双重计分）
- DF 硬门槛从一刀切（DF<4 封顶 5.0）改为两档（DF≤3 封顶 5.0，DF=4 封顶 6.0）

## 6 个维度

### TS — Title Searchability（标题搜索性，权重 20%）

*标题能否被目标观众搜到？搜索需求强度如何？是否包含工具名+结果承诺？*

**搜索需求强度层级**（影响同档位内的最终落分）：
- **刚需**（安装/配置/入门/必用功能）> **高频**（常用场景/工作流）> **改善型**（进阶功能/锦上添花）> **话题型**（热点/观点）

评分标准：
- **10** — 数字 + 工具名 + 结果承诺 + **刚需场景**（"7个配置让Claude Code快3倍""必装三件套"）
- **8** — 工具名 + 动作词 + 结果（刚需或高频搜索），或 系列编号 + 具体功能（"Claude Code安装教程，3分钟搞定"）
- **6** — 场景 + 结果（高频），或 **改善型**功能描述（"AI帮你读网页点鼠标看屏幕"），或 数字 + 主题（"5只小龙虾分工协作"）
- **4** — 仅主题 + 描述（话题型，"AI圈大火的XX"）
- **2** — 纯主题，无搜索关键词（"家庭新成员"）

### OP — Operability & Effect Verification（可操作性与效果验证，权重 20%）

*口播文字是否描述清楚操作路径？操作后是否展示/描述了效果？*

**三种高分路径（均为教程型核心，不偏袒某一种）**：
1. 提示词/文本型：口播念全文（如 video-02 的 7 个提示词）
2. 安装/操作型：口播描述精确操作路径 + 画面展示命令（如 video-01 的点击路径）
3. 对比/测评型：口播描述对比过程和结果（如 video-05 的 4 组对比）

评分标准（综合操作路径 + 效果验证）：
- **10** — 口播给出完整可复制内容（提示词全文/配置文本）+ 描述前后效果对比
- **8** — 口播描述精确操作路径（"点击XX→找到YY"）+ 画面/文字有展示效果
- **6** — 有流程框架但路径不够具体，或有效果但无对比
- **4** — 仅展示概念流程，无操作细节，无效果描述
- **2** — 纯观点/解说，无任何操作内容和效果

**跟做门槛标注**：若操作依赖翻墙/特定平台(仅Mac/仅Win)/付费工具，在评分说明中标注 `[高门槛: 原因]`。不直接扣分（路径描述质量不变），但作为诊断信号提醒——高门槛会降低实际跟做率和互动数据。

### DF — Duration Fit（时长适配，权重 15%，含硬门槛）

*视频时长是否在教程型黄金区间？时长不足是硬伤。*

**⚠️ DF 评分必须用实际视频时长，禁止用字数估算。**

口播稿字数 ≠ 成片时长。实际数据证明偏差极大：

| 口播稿字数 | 实际成片时长 | 语速 | 偏差 |
|-----------|------------|------|------|
| 900 字 | 3:43（223s） | 4.0 字/秒 | 基准 |
| 800 字 | 2:53（173s） | 4.6 字/秒 | 略快 |
| **1500 字** | **3:41（221s）** | **11.3 字/秒** | **字数高估 2.5 倍！** |
| 700 字 | 3:18（198s） | 3.5 字/秒 | 偏慢（有停顿/B-roll） |
| 460 字 | 2:00（120s） | 3.8 字/秒 | 基准 |

原因：口播加速、剪辑跳剪、B-roll 穿插都会大幅压缩成片时长。1500 字的稿子因为语速极快+跳剪，实际只有 3:41。

**获取实际时长的方法（优先级从高到低）**：
1. **douyin-session 抓取**：`data.stats` JSON 中的 `duration` 字段（毫秒，除以 1000 得秒）
2. **抖音创作者中心**：视频管理页面直接显示
3. **估算（最后手段）**：按 **4 字/秒**（正常语速）估算，但必须标注 `[估算]`。注意此方法对长稿（>1000字）严重偏高，实际时长通常只有估算的 60-70%

**硬门槛（v2.0 两档制）**：
- DF ≤ 3（视频 ≤ 90 秒）→ composite 封顶 5.0
- DF = 4（视频 91-120 秒）→ composite 封顶 6.0
- DF ≥ 5（视频 > 120 秒）→ 正常计算

评分标准：
- **10** — 2-5 分钟，信息密度高
- **8** — 1.5-2 分钟，紧凑但完整；或 5-6 分钟略长但节奏好
- **6** — 5-7 分钟，略长但可接受；或 ≈1.5 分钟刚好达标
- **4** — >7 分钟信息稀释；或 ≈90 秒偏短
- **2** — <90 秒，信息不足

### CV — Collection Value（收藏价值，权重 25%）

*观众有没有动力收藏？是否有现成资源可直接获取？*

评分标准：
- **10** — 有现成资源可直接获取（提示词包/代码/配置/模板）+ 结尾明确说"已打包/已准备好"
- **8** — 有具体可保存的操作参数/配置值/完整提示词/产品对比信息；**或**框架型命名（如"三件套""五步法"）+ 结尾有收藏引导，观众会为"以后用到"而收藏
- **6** — 有值得收藏的方法论/操作步骤/流程框架
- **4** — 纯信息消费，看完即走
- **2** — 无任何收藏动力

**收藏驱动力层级**（评分说明中标注，辅助复盘诊断）：
- **L1 行动前必收藏**："装之前先收藏""先收藏再跟着做" — 观众在执行动作前就需要回来，收藏转化率最高
- **L2 即时可用收藏**："提示词包已打包""代码都准备好了" — 有现成资源可立即使用
- **L3 知识储备收藏**："以后可能用到""感兴趣可以收藏" — 收藏转化率较低，依赖观众自律

### HP — Hook Potential（钩子强度，权重 15%）

*前 3 秒能否留住观众？*

评分标准：
- **10** — 3 秒内：痛点 + 工具名 + 结果承诺
- **8** — 5 秒内：场景代入 + 好奇心（"当你一不小心..."）或 名人/权威背书
- **6** — 有场景但缺乏紧迫感
- **4** — 直接进入主题，无钩子
- **2** — 缓慢铺垫，10 秒以上无实质内容

### EC — Ending Completeness（结尾完整度，权重 5%）

*结尾是否驱动收藏和互动？*

评分标准：
- **10** — 资源钩子（"已打包好了"）+ 收藏引导 + 互动引导 + 具体下一步 + 告别语
- **8** — 资源钩子或收藏引导 + 告别语 + 系列标签
- **6** — 告别语 + 互动引导（缺收藏引导）
- **4** — 仅有告别语
- **2** — 突然结束或无收束

## 发布策略检查清单（独立于 composite，PUBLISH 阶段必检）

> rubric 评内容质量，发布策略评传播条件。两者分开，但都影响最终效果。
> 类似 DF 硬门槛（≤3 封顶 5.0，=4 封顶 6.0）的机制，作用于不同阶段。

| 检查项 | 绿灯 ✅ | 黄灯 ⚠️ | 红灯 ❌ |
|--------|---------|---------|---------|
| 发布时间 | 19:00-22:00 | 12:00-14:00 / 7:00-9:00 | 23:00-7:00（强烈建议改时间） |
| 发布间隔 | 距上次 ≥ 3 天 | 距上次 1-2 天 | 同日连发 |
| 选题节奏 | 教程型为主（60%+）| 教程/观点各半 | 连续 3 条非教程型 |

**硬门槛**：红灯项 ≥ 1 时，即使 composite ≥ 6.0 也应暂停发布，调整后再发。

## 复盘判定标准（RETRO 阶段使用）

> 发布后拿到实际数据时，用以下阈值判断"实际好不好"，对比预测方向是否一致。

### 效果判定（收藏率为核心）

| 指标 | 好 | 一般 | 差 | 来源 |
|------|-------|---------|-------|------|
| 收藏率 | >1.3% | 0.5%-1.3% | <0.5% | 抖音创作者中心 |
| 2s跳出率 | <25% | 25%-35% | >35% | 抖音创作者中心 |
| 传播系数（分享/播放） | >0.5% | 0.15%-0.5% | <0.15% | 抖音创作者中心 |

阈值基于17条自有视频全量分析校准（2026-05-02 贤德数据虾）。

### 复盘结论映射

| 预测方向 | 实际效果 | 结论 | 对 rubric 的意义 |
|---------|---------|------|-----------------|
| 好（>1.3%） | 好（>1.3%） | 方向一致 | rubric 通过，积累为校准样本 |
| 好（>1.3%） | 一般（0.5-1.3%） | 预测偏高 | 标记偏差，重点找哪些维度打高了 |
| 好（>1.3%） | 差（<0.5%） | 严重偏差 | 重点复盘，rubric 可能需要 BUMP |
| 一般（0.5-1.3%） | 好（>1.3%） | 预测偏低 | rubric 可能过于保守，考虑 BUMP |

### 2s跳出率对 HP 维度的回溯

- 跳出率 >35% -> HP 维度打分可能偏高，下次降 1-2 档
- 跳出率 <25% -> HP 维度打分合理，可作为锚点参考

### 数据获取

- **首选**：douyin-session adapter 抓取
- **手动**：Jeffrey 在 Telegram 告知已发布，触发数据抓取
- **时机**：发布后 24 小时采集，T+3 天补采一次

### 验证结论

**Spearman rho ≈ 0.57**（14条数据，中等正相关）

**Tier 分层效果**（按 composite 分层后的平均赞数）：
- Tier 1（≥8.0, 3条数据）：平均赞 ~34,947 — 教程型标杆
- Tier 2（6.5-7.9, 4条数据）：平均赞 ~7,991 — 合格教程
- Tier 3+4（<6.5, 7条数据）：平均赞 ~5,194 — 观点/信息/偏短

Tier 1 是 Tier 2 的 4.4 倍，Tier 2 是 Tier 3 的 1.5 倍。分层有效。

**⚠️ 标记解释**：
- "话题驱动"：抄袭争议(v-06)/访谈话题(v-08,v-09,v-13)靠话题热度获赞，非内容质量
- "情感驱动"：产品揭晓(v-10)靠情感共鸣，非教程价值
- "高质低赞"：高质量教程但赞数偏低，可能受发布时间/话题热度影响

**核心发现**：
1. rubric 测量的是**内容质量**，不是**传播力**。教程型维度（OP/CV/DF）决定质量，传播力还受话题/争议/情感驱动
2. DF 硬门槛生效：v-14(328字/54秒) 和 v-20(313字)、v-29(286字) 被正确封顶
3. 观点型内容(v-27,v-36,v-37)被正确识别为 Tier 4，OP=2、CV=2-4
4. v-05(9.00最高分)和v-01/v-02(8.70)的高分主要来自 OP+CV+EC 三维度均衡，验证了教程型核心指标设计

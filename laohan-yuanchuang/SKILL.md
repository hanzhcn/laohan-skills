# 老韩原创：热点转译选题法

把实时热点翻译成老韩能说的、观众想听的原创选题。

核心思路：**热点 = 选题来源（100%），老韩视角 + style.md = 差异化。** "原创"不是从零生造，是在别人都说同一个热点的时候，你说得不一样。

**转译公式：你的真实 AI 经历 + 热点话题 = 你专属的选题。** 观众要的不是 AI 知识，是 **AI 跟我的关系**。你底层是 AI，但观众听的不是教程，是他们自己的生活怎么被 AI 改变了。

**铁律：标准迁就爆款，不迁就创作者。** 如果一个选题需要真实感受才能做好，那就想办法找到真实感受——从老韩的日常观察、身边的人和事、花钱买来的教训里找。不能因为"我没有亲历过"就降低内容标准或跳过好选题。观众不关心你有没有经历过，他们关心的是你说的话像不像真话。

## 转译案例（理解"转译"到底是什么）

这些案例展示了同一个热点如何被不同人讲出完全不同的东西。别人在"报道热点"，老韩在"翻译成我的经历"：

| 热点 | 别人怎么讲 | 老韩怎么转译 |
|------|-----------|------------|
| 35岁男子被AI取代通知（知乎242万热度）| 报道新闻、分析法律赔偿 | "我就是那个用AI取代别人的人"——OpenClaw 8个agent帮我做内容，一个顶几个人的活。讲我用AI做掉了什么岗位的工作 |
| AI大模型中文比英文更费Token（36Kr）| 技术分析：为什么UTF-8编码更占token | "我月烧50亿token，告诉你中文到底贵在哪"——你是真金白银花过这个钱的人。老兵视角算账：到底值不值 |
| CTO转身去Anthropic当工程师（36Kr）| 行业新闻：高管跳槽趋势 | "老韩在互联网十几年见过几次周期"——不是技术分析，是职业选择分析。产品老兵判断这个趋势意味着什么 |
| 假期出游被AI坑惨了（36Kr）| 罗列AI规划的翻车案例 | "AI帮我规划过旅行/帮我做过XX，坑了我多少钱"或"省了我多少钱"——真金白银的账，数字具体，故事真实 |

**案例揭示的模式：** 转译的关键不是换个说法讲同一件事，是**换成你的身份去讲这件事**。你是花钱的人、是用工具的人、是见过周期的人、是身边有真实故事的人。

## 完整流程

### Step 1：全平台抓热点

用 opencli 批量抓取，尽量覆盖多平台。热点出现在越多平台，说明受众面越广，选题价值越高。

```bash
# 核心平台（必抓）
opencli zhihu hot
opencli weibo hot
opencli 36kr hot
opencli douyin hashtag hot --limit 30    # 注意：不是 opencli douyin hot
opencli bilibili hot

# 补充平台（扩大覆盖面）
opencli toutiao hot
opencli tieba hot
opencli hupu hot
opencli hackernews hot
```

抓取后做**交叉验证**：同一话题出现在 3 个以上平台 → 高价值选题。出现在 5 个以上 → 必做。

**自动筛选规则**（处理 50+ 条热榜）：抓取后按以下规则快速缩减到 3-5 个候选：
1. **第一轮过滤**：排除纯娱乐八卦、体育赛事（除非有 AI/科技角度）、地域性新闻、纯政治新闻
2. **关键词匹配**：标题或摘要含 AI / GPT / Claude / 大模型 / 机器人 / 自动化 / 失业 / 裁员 / 编程 / 副业 / 赚钱 → 保留
3. **间接相关打捞**：含 教育 / 职业 / 消费 / 创业 / 老龄化 / 年轻人 / 职场 / 35岁 → 可能找到 AI 连接点，保留
4. **交叉验证排序**：按出现平台数降序排列，取 top 3-5 进入 Step 2

### Step 2：读热点内容（不能只看标题）

对筛选出的候选热点（3-5个），实际读取内容提炼核心论点：

- 36Kr / 知乎文章 → 用 opencli 或 agent-reach 抓正文
- 微博热搜 → 看前几条高赞评论说了什么
- 抖音热榜 → 看相关视频的标题和标签
- B站热门 → 看视频简介和弹幕关键词

提炼内容时关注：**核心争议是什么？大众情绪偏向哪边？有什么数据支撑？**

### Step 3：转译筛选（三个必答题）

对每个候选热点，必须回答三个问题。三个都能回答 → 选题成立：

1. **跟 AI 有关系吗？** → 直接相关最好；间接相关需要找到连接点（如"35岁被裁"→ AI取代岗位的话题）；完全无关 → 跳过
2. **我有真实经历吗？** → 有最好。没有就从老韩的日常中找到能对应上的——楼下邻居、朋友公司、花过的钱、踩过的坑。实在找不到，就基于老韩的人设（产品老兵、token大户、十几年互联网）构造一个合理的亲身感受，让观众觉得"这人真的经历过"。不能因为没亲历就放弃好选题。
3. **路人为什么要在意？** → 观众不是开发者，他们关心的是：能不能赚钱、会不会失业、省不省钱、好不好用。把技术话题翻译成生活语言。

**兜底**：如果所有候选都 fail 这三个问题，回到 Step 1 扩大筛选范围——放宽间接相关关键词，或看补充平台（toutiao/tieba/hupu）是否有遗漏。

### Step 4：确定角度

从老韩的已知视角中选最强的角度。参考角度库（不限于这些，根据热点灵活匹配）：

| 角度 | 适用场景 | 老韩的优势 |
|------|---------|-----------|
| 就业焦虑 | AI取代岗位、裁员、职业转型 | 你是用AI替代人工的人（OpenClaw 8 agent） |
| 消费/省钱 | AI工具价格、token费用、性价比 | 月烧大量token的真实用户 |
| 教育/成长 | AI学习、能力提升、认知升级 | 从零自学到搭建8 agent系统的亲历者 |
| 职业选择 | 行业趋势、跳槽、创业 | 十几年互联网+AI产品视角 |
| 日常场景 | AI在生活中的应用 | 楼下邻居、朋友公司、日常消费的真实观察 |
| 反常识打假 | AI谣言、过度宣传、虚假承诺 | 花过真金白银试过工具的人 |

**角度选择优先级：反常识性 > 情绪共鸣 > 场景包装潜力。** 如果一个热点无法产生反常识角度，它可能不值得做。

### Step 5：生成两句话标题（≤40字）

标题必须是两句话，用逗号、句号或破折号连接：

**第一句（主题句，15-20字）：** 说清楚发生了什么 + 你的立场。让不看科技圈的人一眼就懂。

**第二句（情绪句，10-20字）：** 用生活化语言拉情绪、反转、或制造悬念。必须有具体画面/数字/场景，禁止抽象概念。

**好的例子（实战验证）：**
- "35岁被AI取代通知，拒绝降薪被开除，法院判了——但我想说句真话"（事实+态度悬念）
- "孩子抱AI聊3小时，真不是AI的错，也许他问人活着有什么意思，你不该吓一跳"（主题+生活场景引用+反转情绪）
- "宇树机器人出国被拆电池，丢人还是牛？我觉得这反而是个好事情!"（冲突+态度反转）

**坏的例子：**
- "也许AI能力等级该是划分人群的时候了" — "能力等级""划分人群"是抽象概念，不是生活语言
- "范式转移""赛道消失" — 念出来不像跟朋友说话
- 只有主题句没有情绪句 — 缺第二句 = 少了一半冲击力

**红线：**
- 总长≤40字，第二句必须存在
- 第二句必须用生活语言（像跟朋友发微信说的一句话）
- 禁止抽象概念（"能力等级""范式""赛道"做比喻时禁用）
- 标题念出来必须顺嘴

### Step 7：按 style.md 扩写

大纲完成后，读取 `~/.openclaw/workspace-writer/knowledge/style.md` 进行扩写。

**关键：扩写时遵循 style.md 的语感和技法，但跳过二创脱敏规则（文件末尾"脱敏""结构错位""比喻替换"相关章节）。** 原创内容不需要"结构错位""比喻替换""论证路径换血"——因为没有原稿要脱敏。

需要遵循的 style.md 部分：
- 固定开场白："你有没有这种感觉，"
- 固定结尾：收束→可操作一步→诱饵→系列→告别语
- 口语化词组、废话感、自称
- 技法工具箱 #1-#18
- 禁区（禁用词、禁用标点）
- 内容底线（生活化规则、核心主题贯穿）

不需要遵循的：
- 二创脱敏全部
- Layer A/B 输入映射
- 验证状态标记

## 写稿态度铁律（2026-05-04 实战总结）

> style.md 管语感和技法，这里只管 style.md 没覆盖的：态度、立场、情绪。style.md 已有的规则（框架命名#18、固定开场白、禁区等）不重复。

老韩不是记者，不是旁观者，是替观众说话的人。

### 五条铁律（style.md 未覆盖）

1. **前15秒必须亮出态度，不只是信息。** style.md 说开场要有冲突，但没说要有立场。"35岁老哥被AI取代"是信息，"但他就是教AI的"是态度（讽刺+反转）。观众在前15秒就要感受到老韩站在哪边，不能等到中段才表态
2. **老韩必须下场表态，不能只当转述者。** 注意：这不是 style.md #1 的"亲自下场"（那是指数据验证），这是指情绪参与。通篇都是"这老哥""我朋友说"= 没态度。至少要有1处老韩自己的判断/骂人/质问
3. **身份反差是最大炸弹。** 如果老韩的立场跟大众预期有冲突（比如"我就是那个用AI替代了人的人，但我说那些公司在骗你"），这就是全篇最值钱的话，要放大不要缩着说
4. **细节要具体到让人愤怒。** 不能只说"他公司也这样"，要"设计部五个人裁到两个，活没少，工资降15%"。数字+画面=愤怒
5. **结尾不能说教。** "AI取代的是不愿意学新东西"是鸡汤，砍掉。换成有攻击性的收刀——"你验证过了吗？他要是拿不出证据，那就是假取代。"

### 爆款自检清单（写完后逐条过）

**标题检查：**
- [ ] 标题是两句话吗？第一句主题+冲突，第二句态度+情绪
- [ ] 第二句用了生活语言吗？有具体画面/数字/场景？没有抽象概念（"能力等级""范式"等）？
- [ ] 标题总长≤40字？念出来顺嘴？给不看科技圈的人看能懂吗？

**内容检查：**
- [ ] 前15秒有态度吗？还是只有信息冲突？只有信息 → 加反转/讽刺/判断
- [ ] 通读完，有没有任何一段是纯叙述没有情绪？有 → 加态度
- [ ] 老韩有没有亲自骂人/质问/下判断？没有 → 加
- [ ] 有没有身份反差可以放大？（老韩的立场 vs 大众预期）
- [ ] 朋友/身边人的故事够具体吗？只有抽象词 → 换成数字+画面
- [ ] 结尾听完是"有道理"还是"说到我心坎里了"？前者 → 改
- [ ] 立场摇摆了吗？（两头都占=没站队）→ 选一边站死
- [ ] 有"你们公司是不是也在搞这套"这种促评论的互动钩子吗？没有 → 加

## 标题破圈规则（数据驱动）

标题是观众决定划走还是停留的第一道门槛。历史数据证明：**标题带技术词 = 只有圈内人点，标题用生活语言 = 破圈。**

两句话标题格式详见 Step 5。以下是通用规则：

- **禁止在标题里放**：claude code、openclaw、skill、token、agent、RAG、Obsidian、MoE 等技术术语（除非是专门给技术圈的教程类内容）
- **标题公式优先级**：反常识+悬念（点击率最高）> 场景+渴望（播放爆发力最强）> 数字+悬念（最稳定）> 冲突+反转
- **场景型标题是最强破圈武器**："饺子馆玩AI秒变skill"（3916播放，超均值80%）——"饺子馆"让所有人都能点进来
- Claude Code / OpenClaw 是你的**秘密武器**，不是标题。标题说"我用AI干了件什么事"，不说"claude code教程"

**标题自检：** 给不看 tech 推送的人看这个标题，他能理解在说什么吗？不能 → 改标题。

**反面教材（真实数据）**：
- "GLM5.1降智真相"（440播放，深夜发+圈内标题双重暴死）→ 标题只有圈内人懂，路人完全无感
- "claude code配置教程"（1771播放）→ 教程类固有天花板，"claude code"三个字限死在技术圈。1771 不是失败，但同精力做场景类可以翻倍
- 技术术语标题的上限瓶颈：圈外人看不懂→不点→算法不给量→播放封顶在 2000 以下

## 前3秒法则

抖音算法看的是完播率和互动率。观众决定停留还是划走，在**前3秒**就完成了。

- **前3秒必须有冲突**：不是"今天我来教你…"，不是慢热铺垫。直接甩出负面痛点、反常现象、或认知冲突
- **固定开场白对齐 style.md**：每篇都以"你有没有这种感觉，"开头，之后立刻接冲突（style.md 强制要求）。"你有没有这种感觉"这8个字说完马上进核心内容，前30秒不超过120字
- **标题关键词在前3秒内必须出现**：标题说"一人公司"，"你有没有这种感觉"之后的第一句就得提到"一人公司"
- **完播率 > 时长**：60秒的视频如果80%的人看完，比180秒只有30%看完的视频获得更多推荐。非教程类目标120-150秒，能用60秒说清的不要拖到120秒

## 发布策略

- **发布时间**：晚高峰 19:00-20:00（均播 3208，是其他时段 1.8 倍）。深夜发布严重拖后腿（GLM5.1 23:54发=440播放）
- **标题类型**：场景型播放爆发力最强，数字型最稳定，热点型效果参差
- **时长**：完播率 > 时长。内容质量比时长重要，精华优先于字数

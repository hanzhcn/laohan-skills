# 禁用措辞黑名单

## dim5 软化措辞（可执行具体性，权重 17）

**禁用**（出现 ≥3 处，dim5 扣 ≥3 分）：
- 建议 / 可以考虑 / 根据情况 / 灵活把握 / 视情况而定 / 酌情 / 适当 / 尽量 / 可能需要 / 视乎

**替换为**：祈使句 + 具体参数/示例。
- ❌ "建议使用 playwright" → ✅ "用 playwright（`npm i playwright`）"
- ❌ "根据情况选择平台" → ✅ "抖音用 A 方法，B站用 B 方法"（具体列出）

## dim7 AI 腔废话（整体架构，权重 12）

**禁用**（一处扣 1 分）：
- 说白了 / 换句话说 / 首先...其次...最后 / 综上所述 / 总而言之 / 毋庸置疑 / 显而易见 / 众所周知 / 不可否认 / 不难发现

**替换为**：直接陈述，去掉过渡废话。

## dim9 危险动作黑名单（反例，权重 6）

skill 中必须**显式列禁**的危险操作：
- `rm -rf`（尤其带变量路径）
- `git reset --hard`（回滚用 `git revert`）
- `git push --force` / `--force-with-lease`（除非明确授权）
- `npm publish` / `gh release create`（需用户确认）
- 写入用户主目录外 / 删除非生成文件
